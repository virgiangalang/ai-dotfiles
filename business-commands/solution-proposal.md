# /solution-proposal

Ask: "Describe the business problem you're solving and who the client/stakeholder is. What systems are currently in place?"

Then produce:

```markdown
# Solution Proposal
Client: [Name] | Date: [Date] | Author: [Name]

---

## Executive Summary
[3–4 sentences: problem, proposed solution, expected outcome, investment range]

---

## Problem Statement
### Current State
[Description of how things work today — specific and concrete]

### Pain Points
1. [Pain 1] — Impact: [quantified if possible]
2. [Pain 2] — Impact: [quantified if possible]
3. [Pain 3] — Impact: [quantified if possible]

### Cost of Inaction
[What happens if this is not solved in the next 6 months?]

---

## Proposed Solution

### Approach
[Non-technical description of what we're building and why]

### How It Works
[Simple flow or component description — accessible to a non-technical executive]

### Architecture Overview
[System components, integrations, data flow — technical summary]

---

## Solution Options Considered

| | Option A | Option B | Option C |
|-|----------|----------|----------|
| Approach | [description] | [description] | [description] |
| Timeline | [est.] | [est.] | [est.] |
```
